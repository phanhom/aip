# AIP Version 1.5.0

**Release Date:** 2026-03-12
**Status:** Current

## Summary

Platform-Side Agent Discovery ‚Äî the platform can register any agent by URL alone, auto-detecting its protocol and handling all translation server-side. No bridge process, no sidecar, no extra install on the agent machine.

## Features

### Platform-Side Agent Discovery (Section 17)

- **Protocol profiles**: standardized definitions for how the platform interacts with different agent types:
  - `aip` ‚Äî native AIP (GET /v1/status + POST /v1/aip)
  - `openai` ‚Äî OpenAI-compatible (GET /v1/models + POST /v1/chat/completions)
  - `a2a` ‚Äî Google A2A protocol (GET /.well-known/agent.json)
  - `anthropic` ‚Äî Anthropic messages API
- **Auto-detection algorithm**: platform probes a URL in order (aip ‚Üí a2a ‚Üí openai ‚Üí health) and stops at the first match.
- **Server-side translation**: platform converts AIP messages to native API calls and vice versa, transparently.
- **Health monitoring per profile**: platform uses protocol-appropriate health probes (e.g., `GET /v1/models` for OpenAI, `GET /.well-known/agent.json` for A2A).
- **Profile extensibility**: platforms may define custom profiles with namespaced identifiers.

### Registration Enhancement (Section 15.4)

- New `protocol` field in registration request: `"auto"` (default), `"aip"`, `"openai"`, `"anthropic"`, `"a2a"`.
- New `protocol_detected` field in registration response.
- New `protocol` field in registration record.
- Handshake validation updated to use protocol discovery instead of assuming AIP.

### OpenAPI Updates

- `AgentRegistration` schema: added `protocol` field (enum: auto, aip, openai, anthropic, a2a).
- `AgentRegistrationResponse` schema: added `protocol_detected` field.
- `AgentRegistrationRecord` schema: added `protocol` field.
- Registration endpoint description updated to reference multi-protocol discovery.
- API version: `1.4.0` ‚Üí `1.5.0`.

### Python SDK (`sdk-python/`)

- New `aip.discovery` module:
  - `discover(url, protocol?, timeout?, credentials?)` ‚Äî async function to probe a URL and detect its protocol.
  - `DiscoveryResult` ‚Äî result object with protocol, capabilities, models, and `to_agent_status()` builder.
  - `DiscoveryError` ‚Äî raised when no protocol can be detected.
  - Built-in probe functions for AIP, OpenAI, and A2A profiles.
- Exported `discover`, `DiscoveryResult`, `DiscoveryError` from `aip.__init__`.
- Version: `1.4.0` ‚Üí `1.5.0`.

### Other SDK Updates

- Go, Java, JS/TS SDKs bumped to `1.5.0` for version alignment.

## Breaking Changes

None. All changes are additive. Agents registered without a `protocol` field default to `"auto"`, which first tries `GET /v1/status` ‚Äî preserving the existing AIP-native flow.

## Example

```bash
# Register an OpenClaw agent ‚Äî no bridge, no sidecar
curl -X POST https://platform.example.com/v1/registry/agents \
  -H "Content-Type: application/json" \
  -d '{ "base_url": "http://192.168.1.10:3000", "namespace": "dev-team" }'

# Response:
# { "agent_id": "openclaw-v1", "protocol_detected": "openai", "status": "active", ... }
```
