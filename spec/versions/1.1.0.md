# AIP Version 1.1.0

**Release Date:** 2026-03-12
**Status:** Superseded by 1.3.0

## Summary

Major protocol expansion: human-facing presentation, multi-tenancy, file transfer, agent onboarding, observability, cost tracking, and the universal bridge CLI.

## Features

### Human-Facing Presentation (Section 5.10)
- `Presentation` object on `AgentStatus`: `display_name`, `tagline`, `icon_url`, `color`, `locale`, `categories`, `homepage_url`, `privacy_policy_url`, `tos_url`, `provider`.
- `Provider` sub-object: `name`, `url`.
- Localization guidelines (BCP 47 locale tags).

### Multi-Tenancy (Section 5.11)
- `namespace` field on `AgentStatus` and `GroupStatus`.
- Logical isolation boundary for multi-tenant platforms.

### File Transfer (Section 6.6)
- `POST /v1/artifacts` — multipart/form-data upload, any file type/size.
- `GET /v1/artifacts/{id}` — download artifact content.

### Agent Onboarding Protocol (Section 15)
- `POST /v1/registry/agents` — one-call registration with handshake.
- Platform probes `GET {base_url}/v1/status` during registration.
- Mutual authentication via `credentials` in registration payload.
- Capability auto-detection (streaming, tasks, artifacts, skills).
- `POST /v1/registry/agents/{id}/heartbeat` — liveness signal with `commands` in ACK.
- `GET /v1/registry/agents` — list/search registered agents.
- `PATCH /v1/registry/agents/{id}` — update URL, credentials, namespace.
- `DELETE /v1/registry/agents/{id}` — graceful deregistration.
- Timeout rules: 3× interval → probe, 10× interval → failed.
- Platform events: `agent.registered`, `agent.failed`, `agent.recovered`, etc.

### Observability & Cost Tracking (Section 10)
- `TraceEvent` schema with 18 standard `trace_type` values.
- `TraceSeverity` levels (OpenTelemetry-aligned).
- `LLMUsage` model: `model`, `prompt_tokens`, `completion_tokens`, `total_tokens`, `estimated_cost_usd`, `duration_ms`.
- `UsageSummary` with `by_model` and `by_agent` breakdowns.
- `POST /v1/traces` — emit trace events (batch).
- `GET /v1/traces` — query trace events.
- `GET /v1/traces/{event_id}` — single event.
- `GET /v1/usage` — aggregated cost summary.

### Task Model Enhancement
- `AIPTask` now includes `trace_id`, `correlation_id`, `parent_task_id`.

### Universal Bridge CLI
- `aip bridge --agent <url> --platform <url> --secret <key>`.
- HTTP, WebSocket, stdio transport auto-detection.
- API format adapters: `generic`, `openai`, `anthropic`, `raw`.
- Auto-registration and heartbeat loop.
- Python SDK: `pip install aip-protocol[bridge]`.

### SDK Updates
- Python, Go, Java, JS/TS updated with `Presentation`, `namespace`, `Skill`, `AuthenticationInfo`.

## Breaking Changes

None. All changes are additive to v1.0.0.
