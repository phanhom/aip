# AIP Version 1.3.0

**Release Date:** 2026-03-12
**Status:** Superseded by 1.4.0

## Summary

Probe endpoint for instant agent recovery, and full SDK synchronization across Go, Java, and JS/TS.

## Features

### Probe Endpoint (Section 15.11)
- `POST /v1/registry/agents/{id}/probe` — platform-initiated health check.
- Enables "click to retry" in dashboards: instant recovery without waiting for heartbeat backoff.
- Platform probes `GET {stored_base_url}/v1/status` with 5-second timeout.
- On success: updates cached status, marks `active`, emits `agent.recovered`, resets heartbeat tracking.
- On failure: returns `422` with `aip/registry/unreachable`, keeps current failed state.
- Idempotent — safe to call on healthy agents (refreshes cached status).
- Rate limiting recommended: max 1 probe per agent per 5 seconds.
- OpenAPI: `ProbeResult` schema added.

### SDK Synchronization
All SDKs updated to v1.3.0 with full type parity:

**Go SDK (`sdk-go/aip.go`):**
- `AIPTask`: added `TraceID`, `CorrelationID`, `ParentTaskID`.
- New types: `GroupStatus`, `RecursiveStatusNode`.
- New constants: `StatusScope` (`self`, `subtree`, `colony`).
- Full observability types: `TraceEvent`, `LLMUsage`, `UsageSummary`, `ModelUsageBreakdown`, `AgentUsageBreakdown`.
- Trace type constants (18 values) and severity constants (6 values).

**Java SDK (`sdk-java/`):**
- `AIPTask`: added `traceId`, `correlationId`, `parentTaskId`.
- New records: `GroupStatus`, `TraceEvent`, `LLMUsage`, `UsageSummary`, `ModelUsageBreakdown`, `AgentUsageBreakdown`.
- Version: `1.0.0` → `1.3.0`.

**JS/TS SDK (`sdk-js/`):**
- `AIPTask`: added `trace_id`, `correlation_id`, `parent_task_id`.
- New types: `StatusScope`, `GroupStatus`, `RecursiveStatusNode`, `TraceType`, `TraceSeverity`, `TraceEvent`, `LLMUsage`, `ModelUsageBreakdown`, `AgentUsageBreakdown`, `UsageSummary`.
- All new types re-exported from `index.ts`.
- Version: `1.0.0` → `1.3.0`.

## Breaking Changes

None. All changes are additive.
