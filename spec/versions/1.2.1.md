# AIP Version 1.2.1

**Release Date:** 2026-03-12
**Status:** Superseded by 1.3.0

## Summary

Production resilience — retry, reconnect, circuit breaker across all transports.

## Features

### Transport Resilience
- **HTTPTransport**: configurable retry with exponential backoff on `send()` (default: 3 attempts, 1s/2s/4s).
- **WebSocketTransport**: auto-reconnect on connection drop. Checks `ws.open` before each send; reconnects with backoff if closed.
- **StdioTransport**: auto-restart crashed subprocess (up to `max_restarts=5`). Detects dead process before each send.
- Shared `_retry()` helper: configurable `max_retries`, `backoff_base`, `backoff_max`, labeled logging.

### Platform Registration Resilience
- Background retry loop with exponential backoff (1s → 60s ceiling).
- Platform outage at startup no longer means "never register."
- Best-effort deregistration on shutdown (`DELETE /v1/registry/agents/{id}`).

### Heartbeat Resilience
- Exponential backoff on consecutive failures (10s → 120s ceiling).
- Recovery logging: "Heartbeat recovered after N failures."
- `CancelledError` handled cleanly for graceful shutdown.

### Gateway Circuit Breaker
- `_AgentBackend` state machine: `CLOSED → OPEN → HALF_OPEN`.
- Opens after 3 consecutive failures.
- 30-second cooldown before allowing a probe request.
- Background health probe loop: periodically reconnects failed agents.
- Successful request resets circuit to CLOSED.
- 503 returned immediately while circuit is OPEN.

### Graceful Shutdown
- Bridge: cancels heartbeat, deregisters from platform, closes transport.
- Gateway: cancels all background tasks, deregisters all agents, closes all transports.

### Tests
- 12 new tests: `test_resilience.py` covering retry, reconnect, restart, all circuit breaker states.

## Breaking Changes

None.
