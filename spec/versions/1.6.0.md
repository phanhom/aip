# AIP Version 1.6.0

**Release Date:** 2026-03-12
**Status:** Current

## Summary

Protocol completeness release — standardizes action payloads, message routing rules, error format, and conformance testing. Closes the remaining gaps for a production-grade protocol.

## Features

### Standard Action Payload Schemas (Section 18)

Defines the RECOMMENDED `payload` structure for all 9 standard AIP actions:

| Action | Key Fields |
|--------|-----------|
| `assign_task` | `instruction` (required), `deliverables`, `deadline`, `budget`, `tools` |
| `submit_report` | `summary` (required), `status`, `artifacts`, `metrics`, `next_steps` |
| `request_context` | `question` (required), `scope`, `format`, `references` |
| `request_approval` | `action_description` (required), `risk_level`, `impact`, `evidence` |
| `user_instruction` | `instruction` (required), `context`, `attachments` |
| `handoff` | `task_id` (required), `reason` (required), `context_summary` |
| `escalate` | `severity` (required), `reason` (required), `suggested_action` |
| `publish_status` | `event` (required), `agent_id` (required), `reason` |
| `request_artifact_review` | Standard artifact reference fields |

Schema enforcement rules: receivers MUST NOT reject unknown fields, senders SHOULD validate, `instruction` is the minimum viable payload.

### Message Routing Specification (Section 19)

Protocol-level rules for how platforms route messages between agents:

- **Hub-and-spoke model**: agents connect to platform, platform routes.
- **Routing algorithm**: direct match → role match → URL override → self-handling → not found.
- **Delivery semantics**: at-least-once, idempotency via `Idempotency-Key`, ordered within `correlation_id`.
- **Namespace isolation**: messages stay within namespace boundaries by default.
- **Callback routing**: HMAC-SHA256 signed callbacks with `X-AIP-Signature`.
- **Timeout and retry**: 120s timeout, 3 retries with exponential backoff for transient failures.

### RFC 7807 Problem Details (Section 20)

RECOMMENDED error format for platform endpoints using `application/problem+json`:

- Maps AIP error codes to RFC 7807 `type` URIs.
- Adds `aip_error_code` and `trace_id` as AIP extensions.
- Backward-compatible: `POST /v1/aip` continues to use `AIPAck` format.
- New platform endpoints MAY use RFC 7807.

### Conformance Test Suite Expansion

Expanded from 5 to 22 tests with two levels:

- **basic** (8 tests): status + messaging fundamentals — minimum for AIP compliance.
- **full** (22 tests): tasks, artifacts, streaming, idempotency, discovery, scopes.

New tests cover: lifecycle enum validation, presentation schema, skills schema, correlation_id echo, idempotency, SSE streaming, error code format, Content-Type, status scopes, health endpoint, assign_task payload.

Outputs a compliance verdict: COMPLIANT, PARTIALLY COMPLIANT, or NON-COMPLIANT.

## Breaking Changes

None. All changes are additive. Action payload schemas are RECOMMENDED, not REQUIRED.
