# AIP Version 1.2.0

**Release Date:** 2026-03-12
**Status:** Superseded by 1.3.0

## Summary

Multi-Agent Gateway — host N agents behind a single port with standard discovery, routing, and isolation.

## Features

### Multi-Agent Gateway (Section 16)
- One process, one port, N agents with independent transport, health, and heartbeat.
- `GET /v1/agents` — discover all hosted agents.
- `GET /v1/agents/{id}/status` — per-agent status.
- `POST /v1/agents/{id}/aip` — per-agent messaging.
- `POST /v1/aip` — route by `to` field in the message.
- `GET /v1/status` — `GroupStatus` aggregating all hosted agents.
- `GET /health` — per-agent health breakdown.

### Gateway Configuration
- YAML config file: `aip bridge --config gateway.yaml`.
- Per-agent: `id`, `url`, `format`, `secret`, `name`, `role`, `namespace`, `tags`, `color`, `icon`, `timeout`, `protocol`.
- Gateway-level: `platform`, `secret`, `namespace`, `port`, `host`, `public_url`, `heartbeat`.
- CLI overrides: `--port`, `--namespace`, `--secret`, etc.

### Isolation Semantics
- Fault isolation: one backend down, others keep running.
- Independent state per agent (task count, error state, lifecycle).
- Namespace inheritance with per-agent override.
- Graceful shutdown: drain → deregister → close.
- Hot reload via SIGHUP (optional).

### Platform Registration
- Each agent registered independently with explicit `endpoints`.
- Platform treats each as a standalone agent.

### Spec & OpenAPI
- Section 16: Multi-Agent Gateway (9 subsections).
- OpenAPI: `/v1/agents`, `/v1/agents/{id}/status`, `/v1/agents/{id}/aip`.
- Edge cases: duplicate IDs, 100+ agents, message routing conflicts.

### Dependencies
- `pyyaml>=6.0` added to `bridge` optional dependencies.

## Breaking Changes

None.
