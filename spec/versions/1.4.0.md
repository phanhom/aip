# AIP Version 1.4.0

**Release Date:** 2026-03-12
**Status:** Superseded by 1.5.0

## Summary

Agent Assignment Protocol — standardizes the distinction between an agent's native profile (self-declared role, skills, tools) and its platform-assigned identity (job description, team, constraints, granted tools).

## Features

### Agent Assignment (Section 5.12)
- New `AgentAssignment` schema: `assigned_role`, `team`, `scope`, `granted_tools`, `granted_skills`, `constraints`, `supervisor`, `priority`, `assigned_at`, `metadata`.
- `AgentStatus.assignment` field — agents carry both their native profile and their platform assignment in a single status document.
- Three delivery mechanisms:
  1. **Registration response** — platform includes `assignment` in `AgentRegistrationResponse`.
  2. **Heartbeat command** — new `assign` command type in heartbeat ACK (Section 15.6.3).
  3. **Dedicated endpoint** — `PUT /v1/registry/agents/{id}/assignment` (Section 15.12).
- Assignment lifecycle: set at registration, updated at runtime, cleared by sending empty payload.
- `granted_skills` are merged with native `skills` for discovery; `constraints` are informational (enforcement is platform's responsibility).

### OpenAPI Updates
- `AgentAssignment` schema added to `components/schemas`.
- `AgentStatus` updated with `assignment` field.
- `AgentRegistrationResponse` updated with optional `assignment` field.
- `HeartbeatAck` formalized with `commands` array (including `assign` type).
- `PUT /v1/registry/agents/{agent_id}/assignment` endpoint added.
- API version: `1.3.0` → `1.4.0`.

### JSON Schema Updates
- `AgentAssignment` definition added to `status.schema.json`.
- `AgentStatus` updated with `assignment` field referencing `AgentAssignment`.

### SDK Updates (all v1.4.0)

**Python (`sdk-python/`):**
- New `AgentAssignment` Pydantic model in `status.py`.
- `AgentStatus.assignment` field added.
- Exported from `__init__.py`.

**Go (`sdk-go/`):**
- New `AgentAssignment` struct.
- `AgentStatus.Assignment` field added (`*AgentAssignment`).

**Java (`sdk-java/`):**
- New `AgentAssignment` record.
- `AgentStatus` record updated with `assignment` field.

**JS/TS (`sdk-js/`):**
- New `AgentAssignment` interface.
- `AgentStatus.assignment` field added.
- Re-exported from `index.ts`.

## Breaking Changes

None. All changes are additive. Existing agents without assignments continue to work — `assignment` defaults to `null`.
