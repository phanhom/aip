# AIP Version 1.6.2

**Release Date:** 2026-03-24
**Status:** Current

## Summary

Python SDK hardening — fixes 6 bugs against the spec, adds SSE streaming client, full endpoint client API, and callback security utilities.

## Bug Fixes

| # | Issue | Fix |
|---|-------|-----|
| 1 | `authority_weight` accepted any integer | Added `ge=0, le=100` validation per spec Section 7.2 |
| 2 | `AIPAck.status` was untyped `str` | New `AckStatus` enum: `received \| queued \| rejected` |
| 3 | `AgentStatus.lifecycle` was untyped `str` | New `Lifecycle` enum: `idle \| starting \| running \| blocked \| degraded \| failed` |
| 4 | `HTTPTransport` didn't inherit `BaseTransport` | Now properly extends the ABC |
| 5 | `send()` / `async_send()` created new HTTP client per retry | Client moved outside retry loop — connection pool reused |
| 6 | Duplicate `Skill` / `SkillDescriptor` classes | `SkillDescriptor` is now an alias for `Skill` |

## New Features

### SSE Streaming Client (`send.py`)

- `send_stream()` — sync generator yielding `SSEEvent(event, data)` objects
- `async_send_stream()` — async generator variant
- Full W3C SSE parser supporting `event:` / `data:` lines, comments, multi-line data
- `SSEEvent` dataclass for typed event consumption

### Endpoint Client API (`client.py` — new module)

Complete client functions for every spec-defined endpoint, each with sync + async variants:

| Category | Functions |
|----------|-----------|
| Status | `fetch_status`, `async_fetch_status` |
| Tasks | `get_task`, `cancel_task`, `send_to_task` (+ async) |
| Artifacts | `upload_artifact` (multipart), `get_artifact` (binary) (+ async) |
| Registry | `register_agent`, `heartbeat`, `deregister_agent` (+ async) |
| Traces | `emit_traces`, `query_traces` (+ async) |
| Usage | `get_usage`, `async_get_usage` |

### Callback Security (`security.py` — new module)

- `sign_callback(body, secret)` — HMAC-SHA256 producing `sha256=<hex>` for `X-AIP-Signature`
- `verify_callback(body, secret, signature)` — constant-time comparison
- `SIGNATURE_HEADER` / `EVENT_HEADER` constants

## Tests

57 tests total (up from 30), all passing. New coverage:

- `AckStatus` enum and `AIPAck` coercion (4 tests)
- `authority_weight` range validation including boundary rejection (4 tests)
- `Lifecycle` enum values (1 test)
- `Skill` / `SkillDescriptor` alias identity (2 tests)
- HMAC-SHA256 signing, verification, tampering, format (7 tests)
- SSE parsing: single/multi events, comments, multi-line data, trailing events (11 tests)

## Backward Compatibility

All changes are backward compatible:

- `AckStatus` and `Lifecycle` are `str` enums — existing string comparisons still work
- `SkillDescriptor` remains importable as an alias
- Existing `send()` / `async_send()` signatures unchanged
- No removed exports
