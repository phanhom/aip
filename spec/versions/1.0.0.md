# AIP Version 1.0.0

**Release Date:** 2026-03-12
**Status:** Current

## Summary

Initial release of the Agent Interaction Protocol.

## Features

### Messaging Protocol (`POST /aip`)
- Unified JSON message envelope with four layers: Protocol, Execution, Governance, Observability
- 11 standard actions: `assign_task`, `submit_report`, `request_context`, `request_artifact_review`, `request_approval`, `publish_status`, `handoff`, `escalate`, `tool_result`, `sync_skill_registry`, `user_instruction`
- Custom action support with `x-<org>/<name>` namespacing
- Standard acknowledgment format (`AIPAck`)
- Human-as-sender compatibility (no protocol branching)

### Status Protocol (`GET /status`)
- Three query scopes: `self`, `subtree`, `group`
- Self-describing agent status with discovery endpoints
- Recursive status trees for hierarchical topologies
- Flat group-wide status for dashboards and monitoring

### Governance
- `authority_weight` for organizational authority representation
- `approval_state` workflow: `not_required` → `waiting_human` → `approved` | `rejected`
- `requires_approval` flag for governed actions
- `constraints` for attaching policy and deadline constraints

### Reliability
- Retry semantics with exponential backoff and jitter
- Configurable timeout, max retries, and backoff parameters
- Idempotency key support
- No retry on 4xx client errors

### Observability
- `trace_id` and `correlation_id` for distributed tracing
- `latency_ms`, `error_code`, `error_message` in message envelope
- Structured logging with pluggable logger support

## Breaking Changes

None (initial release).
